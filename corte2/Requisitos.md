REQUERIMIENTOS FUNCIONALES
Registrar productos: el sistema permite registrar productos, el administrador ingresa nuevos productos al sistema incluyendo una información básica como nombre, precio, descripción y cantidad disponible.
Actualizar inventario: el sistema permite modificar la cantidad de productos disponibles cuando se agrega nuevos productos o se venden manteniendo el inventario actualizado.
Realizar pedidos: el sistema permite ingresar los pedidos realizados por los clientes.
Realizar seguimiento del pedido: el sistema permite conocer el estado del pedido (pendiente en proceso o entregado) facilitando su control y organización.
Registrar clientes: el sistema permite almacenar la información de los clientes como nombre, contacto y datos relevantes para futuras compras.
Consultar productos disponibles: el sistema permite visualizar que productos están en el stock y cuales no, evitando confusiones a la hora de vender.
Generar historial de compras: el sistema permite guardar un registro de compras realizada por cada cliente facilitando el seguimiento y análisis.
Eliminar productos: el sistema permite retirar productos del sistema cuando ya no se encuentra disponibles o simplemente se quieran eliminar del inventario.
Consultar pedidos: el sistema permite visualizar la lista de los pedidos registrados con su información y estado actual.

REQUERIMIENTOS NO FUNCIONALES
USABILIDAD: el sistema debe ser fácil de usar y entendible.
RENDIMIENTO: el sistema debe responder rápido a las acciones del usuario
DISPONIBILIDAD: el sistema debe estar disponible la mayor parte del tiempo
CONFIABLIDAD: el sistema debe manejar información correcta y sin errores
