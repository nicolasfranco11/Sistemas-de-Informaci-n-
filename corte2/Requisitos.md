REQUERIMIENTOS FUNCIONALES
1.	Registrar productos: el sistema permite registrar productos, el administrador ingresa nuevos productos al sistema incluyendo una información básica como nombre, precio, descripción y cantidad disponible.
2.	Actualizar inventario: el sistema permite modificar la cantidad de productos disponibles cuando se agrega nuevos productos o se venden manteniendo el inventario actualizado.
3.	Realizar pedidos: el sistema permite ingresar los pedidos realizados por los clientes.
4.	Realizar seguimiento del pedido: el sistema permite conocer el estado del pedido (pendiente en proceso o entregado) facilitando su control y organización.
5.	Registrar clientes: el sistema permite almacenar la información de los clientes como nombre, contacto y datos relevantes para futuras compras.
6.	Consultar productos disponibles: el sistema permite visualizar que productos están en el stock y cuales no, evitando confusiones a la hora de vender.
7.	Generar historial de compras: el sistema permite guardar un registro de compras realizada por cada cliente facilitando el seguimiento y análisis.
8.	Eliminar productos: el sistema permite retirar productos del sistema cuando ya no se encuentra disponibles o simplemente se quieran eliminar del inventario.
9.	Consultar pedidos: el sistema permite visualizar la lista de los pedidos registrados con su información y estado actual.

REQUERIMIENTOS NO FUNCIONALES
1.	USABILIDAD: el sistema debe ser fácil de usar y entendible.
2.	RENDIMIENTO: el sistema debe responder rápido a las acciones del usuario minimo 3 segundos
3.	DISPONIBILIDAD: el sistema debe estar disponible la mayor parte del tiempo es decir 24/7
4.	CONFIABLIDAD: el sistema debe manejar información correcta y sin errores
